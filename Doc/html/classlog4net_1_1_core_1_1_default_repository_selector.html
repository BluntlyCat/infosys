Detailed Description

The default implementation of the IRepositorySelector interface.

Uses attributes defined on the calling assembly to determine how to configure the hierarchy for the repository.

ILoggerRepository log4net.Core.DefaultRepositorySelector.CreateRepository ( Assembly  repositoryAssembly,
Type  repositoryType 
)

Create a new repository for the assembly specified

Parameters
repositoryAssemblythe assembly to use to create the repository to associate with the ILoggerRepository.
repositoryTypeThe type of repository to create, must implement ILoggerRepository.
Returns
The repository created.

The ILoggerRepository created will be associated with the repository specified such that a call to GetRepository(Assembly) with the same assembly specified will return the same repository instance.

The type of the ILoggerRepository created and the repository to create can be overridden by specifying the log4net.Config.RepositoryAttribute attribute on the repositoryAssembly . The default values are to use the repositoryType implementation of the ILoggerRepository interface and to use the AssemblyName.Name as the name of the repository.

The ILoggerRepository created will be automatically configured using any log4net.Config.ConfiguratorAttribute attributes defined on the repositoryAssembly .

If a repository for the repositoryAssembly already exists that repository will be returned. An error will not be raised and that repository may be of a different type to that specified in repositoryType . Also the log4net.Config.RepositoryAttribute attribute on the assembly may be used to override the repository type specified in repositoryType .
