Detailed Description

An IIdentifierGenerator that returns an Int64, constructed using a hi/lo algorithm.

This id generation strategy is specified in the mapping file as

&lt;generator class="hilo"&gt;
&lt;param name="table"&gt;table&lt;/param&gt;
&lt;param name="column"&gt;id_column&lt;/param&gt;
&lt;param name="max_lo"&gt;max_lo_value&lt;/param&gt;
&lt;param name="schema"&gt;db_schema&lt;/param&gt;
&lt;/generator&gt;

The table and column parameters are required, the max_lo and schema are optional.

The hi value MUST be fecthed in a separate transaction to the ISession transaction so the generator must be able to obtain a new connection and commit it. Hence this implementation may not be used when the user is supplying connections. In that case a SequenceHiLoGenerator would be a better choice (where supported).
