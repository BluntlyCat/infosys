Bind the appropriate value into the given command.

Parameters
commandThe command into which the value should be bound.
commandQueryParametersListThe parameter-list of the whole query of the command.
singleSqlParametersOffsetThe offset from where start the list of IDataParameter, in the given command , for the this SqlCommandImpl.
sessionThe session against which the current execution is occuring.

Suppose the command is composed by two queries. The singleSqlParametersOffset for the first query is zero. If the first query in command has 12 parameters (size of its SqlType array) the offset to bind all IParameterSpecifications, of the second query in the command , is 12 (for the first query we are using from 0 to 11).

re-set the index of each parameter in the final command.

Parameters
singleSqlParametersOffsetThe offset from where start the list of IDataParameter, in the given command, for the this SqlCommandImpl.

Suppose the final command is composed by two queries. The singleSqlParametersOffset for the first query is zero. If the first query command has 12 parameters (size of its SqlType array) the offset to bind all IParameterSpecifications, of the second query in the command, is 12 (for the first query we are using from 0 to 11).

This method should be called before call IBatcher.PrepareCommand.
