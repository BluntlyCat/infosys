override FilterDecision log4net.Filter.StringMatchFilter.Decide ( LoggingEvent  loggingEvent)
virtual

Check if this filter should allow the event to be logged

Parameters
loggingEventthe event being logged
Returns
see remarks

The rendered message is matched against the StringToMatch. If the StringToMatch occurs as a substring within the message then a match will have occurred. If no match occurs this function will return FilterDecision.Neutral allowing other filters to check the event. If a match occurs then the value of AcceptOnMatch is checked. If it is true then FilterDecision.Accept is returned otherwise FilterDecision.Deny is returned.

bool log4net.Filter.StringMatchFilter.AcceptOnMatch
getset

FilterDecision.Accept when matching StringToMatch or RegexToMatch

The AcceptOnMatch property is a flag that determines the behavior when a matching Level is found. If the flag is set to true then the filter will FilterDecision.Accept the logging event, otherwise it will FilterDecision.Neutral the event.

The default is true i.e. to FilterDecision.Accept the event.

string log4net.Filter.StringMatchFilter.RegexToMatch
getset

Sets the regular expression to match

The regular expression pattern that will be matched against the rendered message. If the message matches this pattern then the filter will match. If a match is found then the result depends on the value of AcceptOnMatch.

One of StringToMatch or RegexToMatch must be specified.
