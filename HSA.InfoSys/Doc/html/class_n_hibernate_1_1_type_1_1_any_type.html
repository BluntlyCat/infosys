Detailed Description

Handles "any" mappings and the old deprecated "object" type. /summary> remarks> The identifierType is any NHibernate IType that can be serailized by default. For example, you can specify the identifierType as an Int32 or a custom identifier type that you built. The identifierType matches to one or many columns.

The metaType maps to a single column. By default it stores the name of the Type that the Identifier identifies.

For example, we can store a link to any table. It will have the results

class_name id_col1

Simple, AssemblyName 5 DiffClass, AssemblyName 5 Simple, AssemblyName 4

You can also provide you own type that might map the name of the class to a table with a giant switch statemet or a good naming convention for your class->table. The data stored might look like

class_name id_col1

simple_table 5 diff_table 5 simple_table 4

/remarks>
